# Application Data Integrity Validation

This validation suite stabilizes the current Phase 4.3 application before any Phase 4.4 migration work.

## One-Command Gate

```bash
make validate-application-data
```

The target runs backend compile, backend unit tests, Phase 4.1/4.2/4.3 validators, the full application data validator, frontend TypeScript, frontend lint, and the frontend data-contract manifest validator.

## Backend Command

```bash
cd backend
python3 scripts/validate_application_data.py \
  --mode test \
  --symbols SPY,QQQ,IWM,DJI,AAPL,NVDA,AMD,ARM,SNDK,MU \
  --json-output /tmp/application_data_validation.json
```

Live mode routes quotes to Finnhub and history to Polygon:

```bash
cd backend
python3 scripts/validate_application_data.py \
  --mode live \
  --symbols SPY,QQQ,IWM,DJI,AAPL,NVDA,AMD,ARM,SNDK,MU \
  --json-output /tmp/application_data_validation.json
```

## What Is Validated

- Endpoint HTTP status and content type
- Required fields for Home, Market, Sectors, Watchlist, Stock Detail, Data Sources, Data Usage, and analysis contexts
- Quote quality: finite positive price, finite percent change, valid timestamp
- History quality: adequate bars, ascending timestamps, no duplicates, valid OHLC, non-negative volume
- Stock Detail required sections: support/resistance, trendline, volume, risk, relative strength, stock rating, technical signals
- Controlled partial/unavailable states
- Raw error text detection: traceback, `ProviderRequestError`, `Internal Server Error`, `undefined`, `NaN`, and infinity
- Provider-routing evidence when run in-process
- Field-level failures and per-symbol failures

## Error Contract

Expected provider/data failures are handled centrally. Provider failures return structured unavailable JSON instead of raw provider exceptions:

```json
{
  "status": "unavailable",
  "source_state": "unavailable",
  "category": "no_data",
  "message": "Market data is unavailable for this request."
}
```

Regression coverage exists in `backend/tests/test_api_error_contracts.py`.
